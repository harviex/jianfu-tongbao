# 整治形式主义为基层减负公开通报典型问题数据库 — 项目立项方案

---

## 一、项目概述

### 1.1 项目名称
整治形式主义为基层减负公开通报典型问题数据库系统

### 1.2 项目背景
为深入贯彻落实中央关于整治形式主义为基层减负的工作部署，需要对各级（中央层面和省级层面）公开通报的典型问题进行系统性收集、分类和整理，建立专题数据库，为各级机关开展相关工作提供数据支撑和参考依据。

### 1.3 项目目标
- 建设一个集中收录中央和省级层面整治形式主义通报的专题数据库
- 实现通报的自动采集、智能分类、全文检索和数据可视化
- 提供便捷的查询检索界面，支持按地区、问题类型、时间等多维度筛选
- 建立情况反映通道，方便基层反馈问题

### 1.4 功能定位
| 功能模块 | 说明 |
|---------|------|
| 通报数据库 | 自动采集中央和各省级公开通报，统一存储管理 |
| 多维度检索 | 支持关键词搜索、AI智能检索、按分类/省份/标签筛选 |
| 情况反映平台 | 提供在线反映渠道，基层单位可提交问题报告 |
| 数据统计分析 | 实时统计通报数量、涉及省份、问题类型分布等 |
| 管理后台 | 通报数据管理、反映情况处理、数据导出等 |

### 2.2 技术选型

| 组件 | 技术方案 | 版本 | 选型理由 |
|------|---------|------|---------|
| 前端 | 原生HTML/CSS/JS | - | 零依赖、加载快、易维护 |
| 后端 | Node.js + Express | Node 18.x | 轻量高效、异步IO、适合API服务 |
| 数据库 | PostgreSQL | 18.3 | 支持全文检索(FTS)、数组类型、GIN索引 |
| AI引擎 | Ollama (qwen3.5:9b) | - | 本地部署、数据不出网、中文理解能力强 |
| 网页抓取 | Firecrawl API | - | 结构化提取、Markdown输出、成功率高 |
| 反向代理 | Nginx | - | 高性能静态服务、HTTPS终止、负载均衡 |

### 3.3 Docker Compose 编排（推荐）

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:18-alpine
    container_name: tongbao-db
    environment:
      POSTGRES_DB: notices
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "127.0.0.1:5432:5432"
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER}"]
      interval: 10s
      timeout: 5s
      retries: 5

  api:
    image: node:18-alpine
    container_name: tongbao-api
    working_dir: /app
    volumes:
      - ./server:/app
    ports:
      - "127.0.0.1:3008:3008"
    command: node server.js
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: notices
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD}

  nginx:
    image: nginx:alpine
    container_name: tongbao-nginx
    volumes:
      - ./public:/usr/share/nginx/html:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - api
    restart: unless-stopped

volumes:
  pgdata:
```

### 3.4 Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name tongbao.example.gov.cn;

    # HTTPS重定向 (生产环境)
    # return 301 https://$host$request_uri;

    # 静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://127.0.0.1:3008;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 30s;
    }
}
```

## 四、数据采集与更新机制

### 4.1 采集策略

| 数据源 | 采集方式 | 频率 | 覆盖范围 |
|--------|---------|------|---------|
| 中央通报 | 人民网专题页 (zzxszy.people.cn) | 每日1次 | 全量中央通报 |
| 省级通报 | 各省网站/web_search | 每周1次 | 已接入省份 |
| 新增省份 | 手动配置数据源 | 按需 | 逐步扩展 |

### 4.2 数据处理流程

```
源网页 → Firecrawl提取正文 → URL去重比对 → 写入数据库
                                         ↓
                                   Ollama AI处理
                                   ├── 生成摘要
                                   └── 优化标签
```

### 4.3 采集调度 (Cron)

```cron
# 中央通报 - 每日凌晨4点采集
0 4 * * * cd /app && python3 scraper.py --source=central >> /var/log/tongbao-scraper.log 2>&1

# 省级通报 - 每周日凌晨5点采集
0 5 * * 0 cd /app && python3 scraper.py --source=provincial >> /var/log/tongbao-scraper.log 2>&1
```

### 4.4 数据质量控制
- **去重机制:** 以 source_url 为唯一约束，自动跳过已存在记录
- **分类校验:** 规则引擎 + AI双重分类，提高准确率
- **人工复核:** 管理后台支持手动编辑、删除、补充数据

### 5.2 备份策略

```bash
#!/bin/bash
# 每日备份脚本 — /etc/cron.d/tongbao-backup

BACKUP_DIR="/data/backups/tongbao"
DATE=$(date +%Y%m%d)
RETAIN_DAYS=30

# PostgreSQL备份
docker exec tongbao-db pg_dump -U $DB_USER notices | gzip > $BACKUP_DIR/notices_$DATE.sql.gz

# 清理过期备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +$RETAIN_DAYS -delete

# 远程备份 (可选)
rsync -az $BACKUP_DIR/ backup-server:/backup/tongbao/
```

## 九、总结

本系统已完成核心功能开发和原型验证，具备通报自动采集、智能分类、全文检索和情况反映等完整功能。当前采用轻量架构（Node.js + PostgreSQL + 原生前端），在保证功能完整的同时降低了运维复杂度。

通过Docker容器化部署、自动化备份和监控告警机制，可实现低成本、可持续的长期运营。系统支持渐进式扩展，可根据需要逐步增加数据源、功能模块和用户规模。
